# 八字排盘算命 Skill

用户给出出生年、月、日、时、性别 → 排出四柱八字 → 分析五行旺衰 → 排出大运流年 → 综合论命。

**理论依据**：《滴天髓》刘伯温注 + 任铁樵增注、《四柱命理正源》《四柱独门铁口直断》刘文元、子平八字传统正理。

## 核心原则（必须遵守）

1. **用神从原命局找，不随大运流年变**（《正源》核心立场）
2. **原命局格局不变**，大运流年只影响吉凶程度
3. **月支是衡量全局干支旺衰的唯一标准**（空亡月支亦然）
4. **八字中任何干支可直接作用**（反对"隔柱不作用"）
5. **身旺身弱与当官发财无直接关系**，组合好即能富贵

## 使用方法

### Step 1: 收集用户信息

向用户确认以下信息：
```
请提供：
1. 出生年份（公历/农历？）
2. 出生月份
3. 出生日期
4. 出生时辰（几点几分？不确定则说大概时间）
5. 性别（男/女，影响大运顺逆）
6. 出生地点（省市区，可选，用于真太阳时校正）
```

### Step 2: 调用引擎排盘

```bash
python3 ~/.hermes/skills/bazi-reading/scripts/bazi_engine.py YEAR MONTH DAY HOUR MINUTE GENDER
```

返回 JSON 包含：
- `four_pillars` — 四柱（年柱/月柱/日柱/时柱）
- `day_gan` / `day_wuxing` — 日干（命主）及其五行
- `wuxing_count` / `wuxing_pct` — 五行数量与百分比
- `shenruo` / `shen_qiang` — 身强身弱
- `day_changsheng` — 日干十二长生状态
- `yongshen` — 用神/喜神/忌神
- `kongwang` — 空亡
- `dayun` — 大运（12柱）
- `current_dayun` — 当前大运
- `current_age` — 当前年龄
- `liunian` — 近15年流年
- `shensha` — 神煞（天乙贵人、驿马、桃花、禄神等）
- `liuchong` / `liuhe` — 六冲/六合
- `geju` — 格局

### Step 3: 解读与论命

### 已知陷阱（已踩过的坑）

#### 1. 年柱干支公式

年柱天干/地支计算必须用 `(year - 4) % 10` 和 `(year - 4) % 12`，因为甲子年对应公元4年。

```python
# 正确 ✅
gan_idx = (year - 4) % 10
zhi_idx = (year - 4) % 12

# 错误 ❌
gan_idx = (year - 3) % 10   # 1978→己(5)而非戊(4)
zhi_idx = (year - 3) % 12   # 1978→未(7)而非午(6)
```

**验证**：1984甲子年 → (1984-4)%10=0=甲, (1984-4)%12=0=子 ✓。引擎中有3处使用年干（`get_year_ganzhi`, `_calc_dayun`, `calculate_bazi`中的五虎遁），必须全部修正。

#### 2. 日柱基准日

**1900-01-01 = 甲戌日(gan=0, zhi=10)**。**不是乙亥日(1,11)，更不是甲子日(0,0)。**

验证方法：
- 从已知正确的 **2000-01-01 = 戊午日(gan=4, zhi=6)** 反推36524天 → 1900-01-01 = 甲戌日(0,10) ✅
- 高氏日柱公式直接验证：1978-10-01 基数59+第274天=干支33→**丙申日** ✅

引擎代码中 `get_day_ganzhi()` 的 base 参数：
```python
base_date = datetime(1900, 1, 1)  # 甲戌日 = (gan=0, zhi=10)
gan_idx = (0 + days_diff) % 10
zhi_idx = (10 + days_diff) % 12
```

**⚠️ 验证方法**：用高氏日柱公式交叉验证。例如 1978-10-01 应为 **丙申日**（不是丁酉日！）。

#### 3. 五虎遁月柱天干索引

五虎遁数组（`wuhu_map`）**从寅月开始**（数组索引0=寅月，1=卯月…），不是从子月开始。

```python
# 正确 ✅
month_gan = wuhu_map[year_gan][(month_zhi_idx - 2) % 12]

# 错误 ❌
month_gan = wuhu_map[year_gan][month_zhi_idx]          # 索引偏移
```

#### 4. 时辰地支映射（17:00属于酉时，不是申时）

`get_solar_hour_zhi()` 函数（精确到分钟）正确地把 17:00-18:59 映射到酉时。但以前 `calculate_bazi()` 内部用了一个简化的 `shichen_map` 字典（只按小时索引），把 **hour=17 映射成了 '申'**，导致 17:00-17:59 被错误地归入申时。

**结论**：必须使用 `get_solar_hour_zhi(hour, minute)` 函数来确定时辰，它精确到分钟且边界正确。**不要**在 `calculate_bazi()` 里用简化的 hour-only 映射表。

```python
# 正确 ✅
hour_dizhi_actual = get_solar_hour_zhi(hour, minute)
hour_dizhi = hour_dizhi_actual[0]
hour_zhi_idx = DIZHI.index(hour_dizhi)

# 错误 ❌
shichen_map[17] = '申'  # hour=17 应映射为'酉'
hour_dizhi = shichen_map[hour]
```

#### 5. 丑藏干顺序

**不是** `[己, 辛, 癸]`。

各支藏干：`子=[癸], 丑=[己,癸,辛], 寅=[甲,丙,戊], 卯=[乙], 辰=[戊,乙,癸], 巳=[丙,庚,戊], 午=[丁,己], 未=[己,丁,乙], 申=[庚,壬,戊], 酉=[辛], 戌=[戊,辛,丁], 亥=[壬,甲]`

#### 6. 用神/喜神/忌神逻辑

身弱时，优先取 **正印 > 偏印 > 比肩 > 劫财** 为用神。

**关键**：用神、喜神、忌神三者不可相同。例如身弱取正印为用神时：
- 喜神 = 比劫（辅助用神）
- 忌神 = 正官/七杀/正财/偏财/食神/伤官（克泄耗）

#### 7. 十神阴阳逻辑

十神生成时，同我（比肩/劫财）以外的所有四个分支（我生、我克、克我、生我），阴阳判定条件必须是 `yy == yy2`（阴阳同），**不是** `yy != yy2`（阴阳异）。原代码曾全部写反。

正确对照表：

| 关系 | 五行关系 | 阴阳同 | 阴阳异 |
|------|---------|--------|--------|
| 同我 | 五行相同 | **比肩** | **劫财** |
| 我生 | 日干生他干 | **食神** | **伤官** |
| 我克 | 日干克他干 | **偏财** | **正财** |
| 克我 | 他干克日干 | **七杀** | **正官** |
| 生我 | 他干生日干 | **偏印** | **正印** |

例如：丁(阴火)克辛(阴金) → 我克+阴阳同 → **偏财**（不是正印）。

#### 8. LunarDate 属性名

`lunardate.LunarDate` 的属性名为：`year`、`month`、`day`、`isLeapMonth`

**不是** `lunarYear`、`lunarMonth`、`lunarDay`。

### 引擎架构说明

`scripts/bazi_engine.py` 分为三层：

1. **CLI层**（`main()`）— 解析 argparse 参数，组装 `user_input` 字典
2. **解析层**（`resolve_birth_info()`）— 农历↔公历转换 + 真太阳时校正
3. **计算层**（`calculate_bazi()`）— 四柱排盘、五行统计、身强身弱、用神、大运、神煞

修改计算层逻辑后，一定要在 CLI 层和 `resolve_birth_info()` 返回的字段间保持同步。`calculate_bazi()` 的 `**extra` 参数传递的元数据必须与 `main()` 中合并到 `result` 的字段一致。
